`compliance/israeli-business-compliance-checklist.pdf` (added in a prior commit) lists 3
law areas. Section 1 ("נגישות" / Accessibility, Israeli Standard 5568) has **exactly 6**
checklist items, matching the task title "Israeli compliance fixes (6)" — so this task
implemented all 6 accessibility items. Sections 2 (spam law) and 3 (privacy law) were
audited but are out of scope for this task:

- **Section 3 (privacy law)** — `privacy.html` already existed and is comprehensive
  (data collected, AI processing, storage/security, third-party services, user rights,
  retention, contact). Not touched. One residual gap: it doesn't explicitly mention

  language toggle), so a cookie notice is likely not legally required. Flagging as a
  possible future follow-up, not fixed in this task.

## The 6 fixes implemented (Israeli Standard 5568 / WCAG 2.1 AA)

1. **Accessibility statement page** — new `accessibility.html` (bilingual EN/HE, same
   design system as `privacy.html`/`terms.html`), linked from every page's footer.
2. **Accessibility coordinator with contact** — named in the statement (Erez Cohen,
   `salarix.support@erezults.com`, 7-business-day response commitment).
3. **Accessibility widget** — new bottom-corner ♿ toggle + panel on every page
   (`index.html`, `privacy.html`, `terms.html`, `support.html`, `accessibility.html`):
   increase text size (true page zoom, scales layout+images together), high-contrast
   theme, stop-animations mode. State persists per visitor via `localStorage`
   (`assets/site.js` → `initA11y()`, `assets/site.css` → `.a11y-*` rules).
4. **Alt text on images** — audited all `<img>` tags across the site; all informative
   images already had descriptive `alt`, decorative logos already used `alt=""`. No
   changes needed here.
5. **Color contrast** — `--faint` CSS variable (used for small labels: footer headings,
   captions, timestamps) computed to ~3.06:1 against the page background — below the
   4.5:1 AA minimum. Raised its alpha from `.34` to `.55`, recomputed to ~5:1. `--muted`
   and `--teal` were already compliant (~7:1 / ~10.7:1) — untouched.
6. **Full keyboard navigation** — added a "Skip to content" link (visible on focus) as
   the first focusable element on every page, plus a global `:focus-visible` outline
   style (previously there were no custom focus styles at all — relied on inconsistent
   browser defaults). Widget is fully keyboard-operable (Tab to toggle, Enter/Space to
   activate, Escape closes the panel).
